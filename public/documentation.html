Achievement 2 Project: myFlix

To build the server-side component of a "movies" web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Users can view my top 10 favorite movies, request data about a single movie by title, or about a single genre, or director.

Users may also register their names and are assigned a unique ID number, using npm uuid.

Existing users may update their user names, add or remove their favorite movies, or deregister

Business Logic URL HTTP Method Request body data format Respond body data format
Return a list of ALL movies to the user /movies GET None A JSON object holding data about ALL movies
Return data about a single movie, by title /movies/[title] GET None A JSON object holding data about the movie containing title, year, description, genre, director, stars, imageURL, and whether the movie is featured
Return data about a genre, by name /movies/genre/[name] GET None A JSON object holding data the genre category and a description of the genre
Return data about a director, by name /movies/director/[name] GET None A JSON object holding data about director - name, bio, and birthdate
Allow users to register /users POST A JSON object holding data about the new user to register, structured like:
{
name: 'Stephen Weaver',
favoriteMovies: []
}		 A JSON object holding data about the new user registered - new user name, unique id number and favoriteMovies
Allow users to update their user name /users/[id] PUT A JSON object holding data about the user's name to be updated, by unique id number,
structured like:
{
name: 'Stephen Weaver',
favoriteMovies: []
}		 A JSON object holding data about the user's name updated - updated name, unique id number and favoriteMovies
Allow users to add a movie to their list of favorites /users/id/[favoriteMovies] POST None A text message indicating the user's favoriteMovies title was updated with the new title, structured like:
favoriteMovies has been added to user x
Allow users to remove a movie from their list /users/id/[favoriteMovies] DELETE None A text message indicating the user's favoriteMovies title was removed, structured like:
favoriteMovies has been removed from user x
Allow existing users to deregister /users/[id] DELETE None A text message indicating the user has been deregistered, structured like:
User [id], [name], has been deregistered
